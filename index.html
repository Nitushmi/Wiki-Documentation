2. Development Journey

The Human Resource Management System (HRMS) was developed following a structured approach, ensuring scalability, security, and efficiency. Below is an overview of the technology stack and development phases that shaped the system.

2.1. Technology Stack

The HRMS is built using modern technologies that enhance performance, security, and seamless user experience:

  • Backend: Developed using Node.js - 22.12.0 (Express.js - 4.21.2) for building a robust and scalable RESTful API.
  • Frontend: Built with Next.js - 15.1.6, a React-based framework that ensures fast rendering and enhanced user experience.
  • Database: Utilizes MongoDB (NoSQL database) to store and manage employee records, attendance data, and other HR-related information.
  • Authentication: Implements JWT-based authentication for secure access control and data protection.
  • Deployment: Hosted on multiple platforms, including a local server, Vercel, Render, and DigitalOcean, with documentation managed in Wiki.js for knowledge sharing.
  • CI/CD: Uses Jenkins for automating build, testing, and deployment processes.
  • Version Control: Managed through GitHub, ensuring efficient source code tracking and collaboration.

2.2. Development Phases

The HRMS was developed through a well-defined software development lifecycle (SDLC), consisting of the following phases:

Planning

  • Requirement gathering to understand HR workflow and organizational needs.
  • Designing system architecture for efficient data handling and modular development.

Development

  • Implementing core modules such as Recruitment, Attendance, Payroll, and Employee Benefits.
  • Developing REST APIs for seamless integration between frontend and backend.

Testing

  • Conducting unit testing to validate individual functions and components.
  • Performing integration testing to ensure smooth interaction between modules.
  • Carrying out User Acceptance Testing (UAT) to refine the system based on real-user feedback.

Deployment

  • Local Hosting: Initial testing on a local server and Vercel for frontend previews.
  • Cloud Deployment: Final production deployment on DigitalOcean, ensuring scalability and availability.

Maintenance

  • Regular updates to enhance features and address security vulnerabilities.
  • Performance optimization to improve system responsiveness and efficiency.

This structured development approach has resulted in a highly efficient and scalable HRMS, empowering businesses with a seamless HR automation solution.

3.5. Attendance

The Attendance Section in the HRMS portal is designed to streamline and automate employee attendance tracking, ensuring accurate and real-time records. It includes two core features:

  • Add Attendance ‚Äì Allows employees to mark their attendance by clicking Check-In and Check-Out buttons based on their shift timings.
  • All Attendance ‚Äì Enables HR personnel to view and manage attendance records for all employees in a tabular format.

 

3.5.1. Add Attendance

The Add Attendance feature allows employees to log their attendance for the selected day. Employees can check in and check out based on their assigned shift and update their attendance status. This guide will walk you through the various sections and functionalities of the Add Attendance feature.

  • Employee Name (Read-Only)
    Your name is automatically populated based on your logged-in user credentials. This field is read-only and cannot be edited.
  • Employee ID (Read-Only)
    Your unique employee ID is automatically populated. This field is also read-only.
  • Department (Read-Only)
    Your department is automatically displayed based on your profile information and cannot be edited.
  • Date
    Select the date for the attendance entry using the date picker provided. This is the date for which attendance will be recorded.
  • Shift 1 Check-Out
    A button labeled "Shift 1 Check-Out" lets you record your check-out time for Shift 1. This button will only be active after your check-in time for Shift 1 has been recorded.
  • Shift 2 Check-In
    A button labeled "Shift 2 Check-In" lets you record your check-in time for Shift 2. It will be enabled based on your assigned shift.
  • Shift 2 Check-Out
    A button labeled "Shift 2 Check-Out" allows you to record your check-out time for Shift 2. This button becomes active only after your check-in for Shift 2 is recorded.
     

3.5.2. All Attendance

It is displayed in a tabular format with essential columns such as employee details, shift check-in/check-out times, duration, and attendance status. This feature also includes the option to export the attendance records as a CSV file for further analysis.

Attendance Records Table
The All Attendance page displays the attendance records of all employees in a table format with some sort of columns defining the attendance details of an employee.

Filtering Attendance Records
You can filter attendance records based on specific criteria:

  • Employee Name
    Allows searching by employee name to view their attendance records.
  • Date Range
    Enables filtering the records within a specific date range (e.g., a particular month or week).
  • Department
    Allows filtering the attendance records by department.

Export CSV Option
The Export CSV button allows you to download the attendance records of all employees in a CSV file format. The CSV file will include the same columns displayed in the table, providing a comprehensive, downloadable report for analysis and record-keeping.

To export the records:

  • Click on the Export CSV button.
